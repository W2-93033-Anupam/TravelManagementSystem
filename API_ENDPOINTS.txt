# TRAVEL MANAGEMENT SYSTEM - API ENDPOINTS
Base URL: http://localhost:5000

===============================================
üîç HEALTH CHECK
===============================================
GET /health
- Description: Check if API is running
- Auth: None
- Response: {"success": true, "message": "Travel Management System API is running"}

===============================================
üë§ CUSTOMER ENDPOINTS
===============================================

1. POST /api/customer/register
- Description: Register new customer
- Auth: None
- Body: {
    "full_name": "John Doe",
    "email": "john@email.com",
    "phone": "+91-9876543210",
    "password": "password123",
    "address": "123 Main Street, City"
  }

2. POST /api/customer/login
- Description: Customer login
- Auth: None
- Body: {
    "email": "john@email.com",
    "password": "password123"
  }

3. GET /api/customer/profile
- Description: Get customer profile
- Auth: Bearer Token (Customer)
- Headers: Authorization: Bearer <token>

4. PUT /api/customer/profile
- Description: Update customer profile
- Auth: Bearer Token (Customer)
- Body: {
    "full_name": "John Updated",
    "phone": "+91-9876543211",
    "address": "456 New Street, City"
  }

5. PUT /api/customer/change-password
- Description: Change customer password
- Auth: Bearer Token (Customer)
- Body: {
    "current_password": "oldpassword",
    "new_password": "newpassword123"
  }

6. GET /api/customer/bookings
- Description: Get customer bookings
- Auth: Bearer Token (Customer)

===============================================
üß™ TESTING SEQUENCE
===============================================
1. Start with: GET /health
2. Register customer: POST /api/customer/register
3. Login customer: POST /api/customer/login
4. Get packages: GET /api/packages
5. Book package: POST /api/packages/book (with token)
6. Check bookings: GET /api/customer/bookings (with token)

===============================================
üìä HTTP STATUS CODES
===============================================
200 - OK (Success)
201 - Created (Resource created)
400 - Bad Request (Invalid data)
401 - Unauthorized (No/Invalid token)
403 - Forbidden (Access denied)
404 - Not Found (Resource not found)
500 - Internal Server Error

===============================================
üí° NOTES
===============================================
- All POST/PUT requests require Content-Type: application/json
- Dates should be in YYYY-MM-DD format
- DateTime should be in YYYY-MM-DD HH:MM:SS format
- Phone numbers should include country code
- Passwords should be at least 6 characters
- JWT tokens expire in 7 days (configurable)
